## 什么是 MaiLauncher？

MaiLauncher 是一个功能强大的 MaiBot 实例管理和部署工具，为用户提供直观的图形化界面来管理多个 MaiBot 实例。它具有以下特点：

- **简单易用**：现代化的 Web 界面，一键部署和管理
- **功能强大**：完整的实例生命周期管理和资源管理
- **实时监控**：WebSocket 实时通信和状态更新
- **跨平台**：支持 Windows、Linux、macOS
- **开源免费**：基于开源许可证

## 🎯 产品概述

MaiLauncher 旨在简化 MaiBot 的部署和管理流程，通过现代化的 Web 界面替代复杂的命令行操作，让用户能够轻松地：

- 部署和管理 MaiBot 实例
- 监控实例运行状态
- 管理 Bot 资源和配置
- 与 Bot 进行实时交互

## 🚀 快速开始

### 第一步：安装 MaiLauncher

参考 [安装指南](./installation.md) 完成安装。

### 第二步：启动服务

1. 启动后端服务：
```bash
cd mailauncher-backend
python main.py
```

2. 访问 Web 界面：
浏览器打开 `http://localhost:23456`

### 第三步：部署第一个实例

1. 点击「部署新实例」按钮
2. 选择 MaiBot 版本
3. 配置安装路径和端口
4. 等待部署完成

### 第四步：管理实例

- 在实例列表中查看所有实例
- 使用启动/停止按钮控制实例
- 点击实例名称查看详细信息

## 🔧 常见问题

### 无法启动后端服务

**可能原因**：
- 端口被占用
- Python 环境问题
- 依赖缺失

**解决方案**：
1. 检查端口 23456 是否被占用
2. 确认 Python 版本 >= 3.8
3. 重新安装依赖：`pip install -r requirements.txt`

### 实例启动失败

**可能原因**：
- 配置文件错误
- 端口冲突
- 权限不足

**解决方案**：
1. 检查实例配置文件
2. 确认端口未被其他程序占用
3. 以管理员权限运行

### Web 界面无法访问

**可能原因**：
- 后端服务未启动
- 防火墙阻止
- 网络配置问题

**解决方案**：
1. 确认后端服务正常运行
2. 检查防火墙设置
3. 尝试使用 127.0.0.1:23456

## 🏗️ 架构设计

MaiLauncher 采用前后端分离的架构设计：

### 前端 (mailauncher)
- **技术栈**: Vue 3 + DaisyUI + Tailwind CSS
- **构建工具**: Vite
- **状态管理**: Pinia
- **跨平台**: Tauri

### 后端 (mailauncher-backend)
- **技术栈**: FastAPI + SQLite
- **异步处理**: asyncio
- **WebSocket**: 实时通信
- **日志系统**: 结构化日志
